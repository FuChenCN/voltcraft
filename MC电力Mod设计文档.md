# Minecraft 电力系统 Mod 设计文档

**开发环境：** Minecraft 1.21.x / NeoForge / Java 21  
**版本：** v0.1 草案  

---

## 一、项目定位

### 目标
在现有科技/工业 Mod 的基础上，增加一套**真实电气传输与保护层**。

- 电力**生产**由其他 Mod 负责（格雷科技、沉浸工程等）
- 本 Mod 负责电力的**传输、分配、保护**
- 底层能量单位使用 **FE（Forge Energy）**，保证与主流 Mod 兼容
- 目标用户：科技工业整合包玩家

### 差异化
现有 Mod 的电线基本是"能传就传"，最多有损耗。本 Mod 在此基础上增加：
- 电压等级体系
- 电流自动推算
- 真实保护器件（漏保、空开）
- 接线错误惩罚机制

---

## 二、核心电力逻辑

### 2.1 电缆设计

**外观：** 一根电缆  
**内部逻辑：** 包含三条线路

| 线路 | 作用 |
|------|------|
| 火线 | 输送电能 |
| 零线 | 电流回路 |
| 地线 | 漏电保护 |

玩家正常铺设电缆即可，只在**接线端子和配电柜处**需要区分三根线的连接。

**电缆等级（按 FE/t 传输上限）：**

| 等级 | 额定传输 | 对应场景 |
|------|----------|----------|
| 低压电缆 | 1,000 FE/t | 小功率设备 |
| 中压电缆 | 8,000 FE/t | 普通机器 |
| 高压电缆 | 32,000 FE/t | 大型工业设备 |
| 超高压电缆 | 128,000 FE/t | 顶级设备 |

---

### 2.2 电压体系

底层 FE 不携带电压信息，电压作为**自定义标签**叠加在线路上。

**电压等级：**

| 等级 | 电压值 | 对应电缆 |
|------|--------|----------|
| 低压 | 220V | 低压电缆 |
| 中压 | 10,000V | 中压电缆 |
| 高压 | 35,000V | 高压电缆 |
| 超高压 | 110,000V | 超高压电缆 |

**电压写入流程：**
1. 发电设备输出 FE（无电压标签）
2. FE 进入**变压器**或**储能设备**
3. 玩家在变压器/储能设备上设定输出电压
4. 电压标签写入线路，随 FE 一起传输

---

### 2.2.1 线路（EnergyNetwork）定义

"线路"是本 Mod 的核心数据结构，明确定义如下：

> **一条线路 = 一组连通的同等级电缆 + 其上的端子/保护器件，共享同一个电压标签。**

**边界规则：**
- 线路在以下位置**断开**（即两侧属于不同线路）：
  - 变压器（两侧电压不同，天然边界）
  - 储能设备（输入侧/输出侧分属不同线路）
  - 配电柜的不同输出路（每路是独立线路）
  - 空开/漏保跳闸时（断开后两侧重新计算）
- 线路在以下位置**合并**：
  - 同等级电缆相邻
  - 接线端子直连两段电缆

**电压标签归属：**
- 电压标签是**线路属性**，不是单根电缆属性
- 玩家铺设电缆把两个不同电压的变压器输出端连到一起 → 触发 4.3 电压不匹配跳闸
- 玩家铺设电缆连接两个**相同电压**的变压器输出端 → 视为并联，正常工作（电流由两个变压器按容量分配）

**持久化：**
- 线路本身不存盘，区块加载时由电缆方块实体扫描连通图重建
- 电压标签存于电缆方块实体的 NBT 中，重建时从任一已知电缆继承

### 2.2.2 电压等级与电缆等级强绑定

| 电压等级 | 允许的电缆等级 |
|---------|---------------|
| 220V | 仅低压电缆 |
| 10kV | 仅中压电缆 |
| 35kV | 仅高压电缆 |
| 110kV | 仅超高压电缆 |

变压器输出端必须连接对应等级的电缆，否则放置时拒绝连接（红色提示）。这样可以避免"用高压电缆跑 220V"或反过来烧毁线路的混乱情况。

### 2.2.3 变压器物理模型（功率守恒）

变压器遵循**功率守恒**原则，FE/t 是功率单位，因此：

```
输入 FE/t × (1 - 损耗率) = 输出 FE/t
```

电压变化只影响**电流推算**，不影响 FE 数值本身。

**示例：** 220V 侧输入 2200 FE/t（推算电流 10A），经过升压变压器到 10kV 侧：
- 输出 FE/t ≈ 2178 FE/t（按 1% 损耗）
- 推算电流 ≈ 0.218A
- 功率守恒：220×10 ≈ 10000×0.218

**损耗率（默认值，可配置）：**
- 普通变压器：1%
- 高效变压器（后期合成）：0.2%

### 2.3 电流自动推算

电流不需要玩家手动设置，系统自动计算：

```
电流（A） = 当前传输 FE/t ÷ 线路电压
```

**示例：**
- 线路电压：220V
- 当前传输：2,200 FE/t
- 自动推算电流：10A
- 若电缆额定 8A → 过载，触发空开跳闸

玩家可通过**钳形电流表**读取各段电流数值。

## 三、漏电触发条件

以下行为会导致漏电，漏电由**玩家操作不当**引起，不会随机发生：

| 触发条件 | 说明 |
|----------|------|
| 电线暴露在雨水/水源旁 | 未套管的电缆接触水 |
| 电线无套管直接埋地 | 地面铺设未使用套管保护 |
| 超额定功率长时间运行 | 持续过载 |
| 接线端子未正确连接 | 火零地接错或漏接 |

---

## 四、跳闸机制

### 4.1 漏保跳闸（漏电保护器）

**原理：** 检测火线与零线的电流差

- 正常状态：火线电流 = 零线电流
- 漏电状态：火线电流 > 零线电流（电流从地线泄漏）
- 阈值：电流差超过 **max(当前线路传输量 × 0.5%, 30mA 等效值)**，避免低电流场景下浮点误差误跳
- 响应：**立刻跳闸**，无缓冲

**地线未接：** 漏保失效，漏电不触发跳闸，但漏电持续存在（见漏电后果）

### 4.2 空开跳闸（空气开关）

**过载跳闸（热脱扣模拟）：**
- 电流超过额定值 120% 持续 **100 tick（5秒）** → 跳闸
- 电流超过额定值 200% → **立刻跳闸**

**短路跳闸（电磁脱扣模拟）：**
- 火线直接接地线 → 立刻跳闸
- 火线与零线直接相连 → 立刻跳闸

### 4.3 电压不匹配跳闸

- 不同电压的电源并联到同一线路 → 立刻跳闸
- 跳闸的是两个电源各自的空开
- 无缓冲

---

## 五、漏电后果

### 已安装漏保（正常情况）
漏保跳闸 → 线路断电 → 玩家排查修复 → 合闸

### 未安装漏保
- 附近玩家和生物持续受到微量伤害
- 漏电处产生**火花粒子效果**
- 不会烧毁机器

### 5.1 触电伤害数值（按电压指数缩放）

| 电压等级 | 每 tick 伤害（半心 = 1.0） | 作用半径 |
|---------|---------------------------|---------|
| 220V | 0.05 | 1 格 |
| 10kV | 0.5 | 2 格 |
| 35kV | 2.0 | 3 格 |
| 110kV | 8.0（直接致命级别）| 4 格 |

- 伤害类型为自定义 `DamageSource`（可被绝缘装备减免）
- 高压区域必须接地，否则不允许玩家放置变压器输出（强制接地桩前置）
- 数值在 ModConfigSpec 中可调

### 5.2 长时间过载与跳闸的关系

- **空开过载跳闸**（4.2 节）是设备保护，针对线路本身
- **长时间过载漏电**（第三章触发条件）是绝缘老化模拟：电流持续 >100% 额定值且**未触发空开**（例如电流刚好在 100%~120% 之间）超过 **1200 tick（60 秒）** 后，该段电缆进入"绝缘损坏"状态，开始向外漏电
- 绝缘损坏的电缆需要拆除重铺，无法自然恢复

## 六、排查工具

跳闸后线路断电，玩家需要用工具定位漏电点：

### 工具层级

| 工具 | 使用时机 | 功能 |
|------|----------|------|
| 验电笔 | 修复后确认 | 检测线路是否有电，最基础工具 |
| 钳形电流表 | 正常运行时预防 | 检测各段电流，发现异常提前处理 |
| 兆欧表 | 跳闸后断电排查 | 测绝缘电阻，精确定位损坏段 |

### 完整排查流程

```
平时用钳形表监测电流
    ↓ 发现异常及时处理
    ↓ 未处理 → 跳闸断电
用兆欧表逐段排查
    ↓ 找到漏电点
修复绝缘/重新接线
    ↓
用验电笔确认线路正常
    ↓
合闸恢复供电
```

### 视觉反馈
- 漏电位置产生**火花粒子**
- 漏保面板显示"已跳闸"状态
- 电气柜显示各路电流状态

## 八、接线规范

### 接线端子的玩家交互

接线端子是一个独立方块，正面有三个连接点：

```
 ┌───────────────┐
 │  [L]  [N]  [E]│   L=火线 N=零线 E=地线
 │   ●    ●    ● │   每个点接受一根电缆
 └───────────────┘
```

- 玩家用**电工钳**右键端子的某个连接点，再右键目标电缆 → 建立逻辑连接
- 端子方块实体记录三个连接点各自连到了哪根电缆（BlockPos）
- 渲染时三根细电缆从端子伸向目标方块（视觉上仍是一根总电缆从机器出来）
- 接错的判定见下表

### 正确接线（接线端子处）
- 火线 → 火线端子
- 零线 → 零线端子
- 地线 → 地线端子/接地桩

### 接线错误后果

| 错误类型 | 后果 |
|----------|------|
| 地线未接 | 漏保失效 |
| 零线接错 | 漏保误跳或不跳 |
| 火线接错 | 直接短路，空开跳闸 |
| 火零地全部接错 | 短路+设备无法工作 |

---

## 九、接地系统

- 每套线路需要至少一个**接地桩**
- 地线必须连接到接地桩，漏保才能正常工作
- 接地桩放置在地面方块上
- 地线断开或接地桩未放置 → 漏保失效

---

## 十、零线回路

- 零线断开 → 电流无法回到电源 → 线路停止工作
- 零线与火线直接相连 → 短路，空开跳闸
- 零线接到地线端子 → 漏保误跳

## 十二·五、区块加载与跨 Mod 兼容

### 区块加载策略

电力网络可能横跨多个区块，必须明确未加载区块的处理方式：

- **被动策略（默认）：** 未加载区块内的电缆视为线路断点，已加载部分自动收缩为子线路；区块重新加载时由方块实体扫描自动合并
- **不主动 chunk-load：** 不强制加载未加载区块（避免 TPS 灾难和与 FTB Chunks 等区块加载 Mod 冲突）
- **边界缓存：** 线路在区块边界处缓存最后已知电压标签，重新连通时校验是否一致；不一致按"电压冲突"跳闸处理

### 与第三方 Mod 的 FE 衔接

**外部 FE 流入本 Mod 电缆：**
- 其他 Mod 的发电机直连本 Mod 电缆 → 拒收，必须经过变压器写入电压标签
- 变压器低压侧设计为**FE 输入端**（无电压要求），高压侧才是带电压的输出
- 这样玩家可以：第三方发电机 → 变压器输入端（普通 FE） → 变压器输出端（带电压标签）→ 本 Mod 电缆

**本 Mod FE 流向第三方机器：**
- 经过**接线端子或下压变压器**降为普通 FE 后输出给第三方机器
- 第三方机器无视电压标签，正常消耗 FE
- 直连高压电缆 → 设备过载烧毁（视觉效果 + 物品损坏）

## 十二·六、配置项（ModConfigSpec）

所有数值阈值通过 `ModConfigSpec` 暴露，整合包作者可调整：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `transformer.lossRate` | 0.01 | 普通变压器损耗率 |
| `transformer.efficientLossRate` | 0.002 | 高效变压器损耗率 |
| `rcd.thresholdPercent` | 0.005 | 漏保电流差百分比阈值 |
| `rcd.thresholdMinAmp` | 0.03 | 漏保最小绝对值阈值（A） |
| `breaker.overloadFactor` | 1.2 | 空开过载系数 |
| `breaker.overloadDurationTicks` | 100 | 过载跳闸延时 |
| `breaker.shortCircuitFactor` | 2.0 | 立即跳闸系数 |
| `damage.220v` ~ `damage.110kv` | 见 5.1 | 各电压触电伤害 |
| `cable.insulationDamageTicks` | 1200 | 绝缘损坏触发时间 |
